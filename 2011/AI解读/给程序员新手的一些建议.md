# 内容分析报告

## 一、核心内容
### 1. 核心论点
在校生简历项目很 NB、算法尚可但编码极弱，根源是缺思考、少实践、被动学习；应主动反思、多写程序、追求程序员而非码农，并利用网络资源弥补学校教育缺失，实习与正式招聘同等严肃。

### 2. 关键概念
| 概念 | 文中定义/内涵 |
|------|--------------|
| NB 项目 | 简历上高深课题多，与真实编码能力脱节 |
| WTF 编码能力 | 算法能口述却写不出 30 行代码 |
| 码农 vs 程序员 | 按部就班写代码 vs 反思、精益求精的创作 |
| 实习生招聘 | 实质等同毕业生招聘的能力筛选 |

### 3. 文章结构
陈皓实习生招聘观察 → 三点学生特点（项目、算法、编码）→ 四条建议（思考、实践、反思、主动学习）→ 说明实习计划即正式招聘通道。

### 4. 案例与证据
- 5% 无提示解出算法，70% 给提示可解，10% 仍不行
- 10 分钟写不出「整型数组最小正数」
- 对比作者当年 Java Applet 课件与现今内核/ML 项目
- 学校不教 Git、TDD、重构等但资源现成

## 二、背景语境
### 1. 作者身份
陈皓，以招聘者视角写酷壳，新浪微博 @左耳朵耗子。

### 2. 写作背景
2011 年实习生计划；国内毕业生量大质次；与《为什么中国的网页这么烂》教育批判呼应。

### 3. 写作意图
警醒在校生；解释公司为何用实习筛选；推动自主学习和编码练习。

### 4. 底层假设
算法题可部分预测工作表现；编码能力应成为硬门槛；学生有时间访问互联网优质资源。

## 三、批判性审视
### 1. 主要反对意见
算法面试与日常 CRUD 脱节；项目 NB 可能来自导师；性别与资源不平等；「犯罪」措辞过重。

### 2. 论证漏洞
样本为一家公司招聘；未区分 CS 与非科班；10 分钟写代码时间压力敏感。

### 3. 适用边界
适合目标进入一线互联网的学生；对非编码岗位不适用。

### 4. 回避问题
课程体系改革责任；心理健康；如何教 Version Control 在学校落地。

## 四、价值提取
### 1. 可复用思考框架
「学完问为什么」三连；「口述→默写代码」自检；实习当正式面试准备。

### 2. 对从业者的启发
维护 GitHub 可运行小作品；每日一题+实现；读经典并写读后感强化记忆。

### 3. 对决策者的启发
实习别当廉价劳动力；给 mentor；招聘 bar 与培养投入一致。

### 4. 认知升级点
知道≠会写；编程是创作学科，需刻意练习。

## 五、观点辩论

### 核心观点列表（≤5）
1. 国内很多学生算法理解尚可但动手编码能力严重不足。
2. 被动接受知识而不思考「为什么」是核心学习缺陷。
3. 实习招聘与应届生招聘在能力要求上本质相同。

### 观点一深度辩论
**[算法思路能讲清但代码写不出，说明教学与评价过度偏向理论作业]**

**第一轮**
- 支持方：招聘数据与写最小正数案例；项目名不副实。
- 反对方：白板压力、环境陌生；应考项目代码质量而非小题。
- 综合方：理论+实现双轨评价；课程强制配套编程量。 | 共识进度：🟢

**辩论结论**：共识表述——「面试与教学都应考核可运行代码，而非仅口述算法。」

### 观点二深度辩论
**[学生「不学习」主因是被动习惯而非教育资源匮乏]**

**第一轮**
- 支持方：Git/经典书/开源在线；时代资源远胜 286 年代。
- 反对方：城乡差异、外语、设备；课程排满无时间；心理健康。
- 综合方：资源不均存在，但主动学习习惯仍是最大杠杆；社会应降低门槛。 | 共识进度：🟡

**辩论结论**：分歧定性——个体能动 vs 结构性不平等；综合表述——「主动学习是关键，同时需承认资源与课程结构障碍。」
