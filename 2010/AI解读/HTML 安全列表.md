# 内容分析报告

## 一、核心内容
### 1. 核心论点
heideri.ch/jso/ 站点系统罗列 HTML5 时代各浏览器中的 XSS 等安全问题，多数与 JavaScript 解析/注入相关，是 Web 开发者的实用攻击面清单；作者体感 IE 与 Firefox 问题相当，Safari 相对较少。

### 2. 关键概念
| 概念 | 文中定义/内涵 |
|------|--------------|
| HTML5 浏览器安全问题集 | 针对新兴 HTML5 特性的跨浏览器漏洞样本库 |
| XSS（跨站脚本） | 通过畸形标签/属性/编码执行脚本 |
| 编码绕过 | 如 HTML 实体、mac-farsi 字符集、getter/setter 语法 |
| javascript: 协议 URL | 在 src 等属性中执行脚本的经典向量 |

### 3. 文章结构
推荐总站点→列举 6 类代表性 payload 与受影响浏览器→指出站点持续更新→粗略对比 IE/Firefox/Safari 安全问题密度。

### 4. 案例与证据
1. table background=javascript:alert(1) — IE6-9、Opera 8-10。
2. meta charset=mac-farsi 畸形 script — 全 Firefox XSS。
3. script 内 HTML 实体编码 — Firefox/Opera/Chrome。
4. Firefox getter/setter 表达式 XSS。
5. IE expression() in style — IE5.5-9。
6. embed/img/script src=javascript: — 多浏览器。

## 二、背景语境
### 1. 作者身份
陈皓，酷壳；安全资讯聚合，非原创漏洞研究。

### 2. 写作背景
HTML5 草案期（2010），浏览器实现不同步；XSS 防御尚未标准化 CSP 普及；开发者急于采用新标签。

### 3. 写作意图
提醒中文 Web 开发者关注 HTML5 **实现差异**带来的安全坑；提供可跟进的外部资源。

### 4. 底层假设
黑名单式 payload 学习对防御有效；浏览器版本与文中所列仍相关（今天部分已修复）；Javascript 是问题核心载体。

## 三、批判性审视
### 1. 主要反对意见
黑名单过时快；未给系统性防御（输出编码、CSP、HttpOnly）；浏览器排名主观；HTML5 名头部分向量实为老 IE 行为。

### 2. 论证漏洞
未验证每条 payload 在 2010 年环境中的复现；Safari「最少」无数据；未区分规范缺陷 vs 实现 bug。

### 3. 适用边界
成立：安全意识启蒙、历史 payload 博物馆入口。失效：现代渗透测试 playbook（需更新）。

### 4. 回避问题
框架（React 等）时代防御；服务端模板责任；合规与 responsible disclosure。

## 四、价值提取
### 1. 可复用思考框架
**新特性安全清单**：每增一 HTML5 API/属性，问「在哪些浏览器可被当作脚本入口？」

### 2. 对从业者的启发
勿信「HTML5 更安全」口号；输入/输出上下文分离；跟踪 jso 类聚合站但要以 CSP 和框架默认为主防線。

### 3. 对决策者的启发
安全发布应含浏览器矩阵测试；禁止内网应用「仅支持 IE」忽视其他引擎差异。

### 4. 认知升级点
同一「标准」在不同引擎是不同攻击面；安全研究站点常比规范草案跑得更快。

## 五、观点辩论

### 核心观点列表（≤5）
1. HTML5 普及会扩大 XSS 攻击面而非自动缩小。
2. 维护外部 payload 列表对团队防御仍有价值。
3. Safari 在 2010 年相对更安全。

### 观点一深度辩论
**[HTML5 扩大 XSS 攻击面]**

**第一轮**
- 支持方：新标签/属性/编码路径增加；文中多例涉及 HTML5 语境。
- 反对方：HTML5 也带来语义化与 CSP 等防御工具；许多向量是遗留 IE。
- 综合方：**过渡期**攻击面扩大（实现未完成+开发者误用）；成熟后取决于部署 CSP 与弃用危险 API。 | 共识进度：🟢

**辩论结论**：共识表述——2010 年语境下成立；长期看取决于防御实践而非标签版本号。

### 观点二深度辩论
**[payload 列表仍有团队价值]**

**第一轮**
- 支持方：培训、回归测试、WAF 规则种子。
- 反对方：维护成本高、易过时、可能鼓励黑名单思维。
- 综合方：作**红队素材库**有价值，须配合白名单/CSP 策略，不可替代架构防御。 | 共识进度：🟢

**辩论结论**：共识表述——列表是补充资源，非防御体系核心。

### 观点三深度辩论
**[Safari 相对更安全]**

**第一轮**
- 支持方：作者体感与样本少。
- 反对方：样本偏差；WebKit 同样历史漏洞多；未含移动端。
- 综合方：2010 年桌面 Safari 份额与攻击关注度低可能导致**观测偏差**，非本质更安全。 | 共识进度：🟢

**辩论结论**：观点修正——「Safari 问题少」应改为「当时攻击样本与关注度在 Safari 上较少」，不宜推广为引擎安全排序。
