## 一、核心内容
### 1. 核心论点
C++ 构造函数涉及大量易错点与惯用法，应通过 Marshall Cline 的 C++ FAQ Lite 构造函数章节系统学习，而非仅凭直觉编码。

### 2. 关键概念
| 概念 | 文中定义/内涵 |
|------|--------------|
| C++ FAQ Lite | parashift.com 权威 FAQ，中文译本止于 2001 过旧 |
| Most vexing parse | List x() 可能被解析为函数声明 |
| 初始化列表 vs 赋值 | 成员应用初始化列表更高效、更安全 |
| Named Constructor Idiom | 用静态成员函数代替多构造函数 |
| static initialization order fiasco | 静态对象初始化顺序未定义导致崩溃 |

### 3. 文章结构
引言挑战读者自测 → 列出 19 个 FAQ 标题与中文翻译链接（答案在英文站）→ 说明中文版老旧 → 鼓励浏览其他章节。索引/导读帖，非答案正文。

### 4. 案例与证据
- 19 条直链：ctors.html#faq-10.1 至 10.19。
- 涵盖构造失败、Named Parameter、Foo x(Bar()) 错误等经典题。
- 注明中文 FAQ 仅更新至 2001。

## 三、批判性审视
### 1. 主要反对意见
只列标题不提供中文答案，完成度低；C++11 后 move、委托构造、=default 等未覆盖；应推荐 cppreference 与《Effective C++》。

### 2. 论证漏洞
无作者原创解析；「你能回答吗」无自测反馈机制。

### 3. 适用边界
适合作为书签索引；系统学习需配合现代 C++ 资料。

### 4. 回避问题
未提 RAII 与智能指针如何改变构造失败处理；未提异常安全等级。

## 四、价值提取
### 1. 可复用思考框架
**C++ 构造问题分类学**：语法歧义 → 初始化语义 → 静态生命周期 → 失败处理 → 惯用法（Named ctor）。

### 2. 对从业者的启发
遇到链接错误先查 static 成员定义与初始化顺序；优先初始化列表；警惕 most vexing parse。

### 3. 对决策者的启发
遗留 C++ 项目培训应包含 static fiasco 与构造异常安全模块。

### 4. 认知升级点
构造函数不是「顺便赋值」，而是对象不变式建立与资源获取的入口。
